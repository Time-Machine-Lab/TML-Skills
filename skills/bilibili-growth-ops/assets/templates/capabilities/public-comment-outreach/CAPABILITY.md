# 功能包：公开评论触达

## 用途

对目标视频进行公开评论触达，形成第一轮引流动作。

## 何时使用

- 已经完成视频筛选，准备进入公开触达阶段时使用
- 当前阶段目标是对视频建立第一轮自然评论入口时使用
- 已经拿到阶段审核结果，允许执行首轮外发时使用

## 前置条件

- 产品资料、按提炼指引整理过的 `PRODUCT-INSIGHT.md`、视频列表和本轮评论方向已经明确
- 当前阶段已通过“先审核后执行（`review-first`）”要求，或该阶段已获得继续执行权限
- 外发前已经做过去重检查，避免重复对同一目标发送相似评论
- 外发前需要调用 `records.cooldown-check`，以返回的近期动作记录和当前生效规则为准，再决定能否继续发送
- 如果本轮需要调整节流规则，应先由主 Agent 调用 `records cooldown-policy-set` 更新中心规则，而不是在这里临时写死数值

## 输入

- 产品标识 slug
- 已通过审核的阶段执行方案
- 目标视频列表
- 评论草稿
- 可选的风险说明或冷却限制

## 产出

- 评论发送结果
- 失败或跳过记录
- 对应的关键动作记录 `OperationRecord`

## 执行步骤

1. 逐个确认目标视频是否适合本轮公开触达。
2. 在发送前先做去重检查，判断该账号是否已经对该目标执行过相似评论。
3. 再做节流检查，以 `records.cooldown-check` 返回的近期记录和当前生效规则判断当前是否适合继续发送。
4. 如果目标可执行，再准备更自然的评论文案，不要机械复读。
5. 发送前补一段简短说明，写清楚本次评论为什么发、目的是什么、依据是什么；最好控制在 100 字内。
6. 确认当前阶段已通过审核后，再执行评论发送。
7. 对发送成功、跳过、失败三种情况都分别记录清楚。
8. 发送完成后，把结果和风险提示交回主 Agent，不在这里直接推进下一个阶段。

## 判断规则

- 如果命中去重记录，应优先跳过，而不是强行重复触达。
- 如果 `records.cooldown-check` 显示当前账号距离上一次外发过近，或当前窗口内次数过多，应暂停并等待，不要硬发。
- 评论要像自然参与讨论，不要直接变成硬广或堆砌卖点。
- 如果阶段审核未通过，不得擅自执行真实外发动作。
- 如果视频上下文不适合评论引流，应宁可少发，也不要硬发。

## 允许使用的指令

- `records.dedupe-check`
- `records.cooldown-check`
- `comment.send`

## 禁止事项

- 不要绕过阶段审核直接外发。
- 不要绕过节流检查直接外发。
- 不要连续对多个目标发送高度相似的话术。
- 不要在评论里直接做强导流或过度营销。

## 完成标志

- 本轮目标视频已经完成评论发送、跳过或失败记录。
- 每一条真实发送都已落关键动作记录 `OperationRecord`，并带有简短说明。
- 已经把本轮执行结果和后续注意事项交还主 Agent。

## 受阻处理

- 如果阶段审核未通过，立即停止执行并交回主 Agent。
- 如果命中风控、权限或发送失败异常，记录目标与原因后停止本轮外发。
- 如果评论文案明显不自然或不符合场景，先暂停并请求重新审阅。
