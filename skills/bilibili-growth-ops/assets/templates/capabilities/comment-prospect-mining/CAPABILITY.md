# 功能包：评论区线索挖掘

## 用途

在指定视频评论区中发现更有意向的用户与线索。

## 何时使用

- 已经有目标视频，需要进一步从评论区发现意向用户时使用
- 希望找到比“广撒网评论”更精准的跟进对象时使用
- 下游阶段需要一份更聚焦的线索列表时使用

## 前置条件

- 已经有明确的视频目标，且视频与产品相关
- 产品资料里已经能说明目标用户、核心卖点与线索判断标准
- 当前阶段目标是“识别线索”，不是直接外发

## 输入

- 产品标识 slug
- 视频 id
- 评论扫描数量上限
- 可选的扫描策略，例如页码、模式

## 产出

- 线索用户列表
- 值得关注的评论
- 后续跟进建议

## 执行步骤

1. 先读取产品资料，优先参考按提炼指引沉淀出的 `PRODUCT-INSIGHT.md`，明确什么样的评论和用户更可能有意向。
2. 再读取视频信息，确认这个视频为什么值得继续挖线索。
3. 扫描主评论区与重点回复链，不只看最热评论。
4. 识别更接近目标用户问题、痛点、求助、比较、求推荐等表达。
5. 把值得继续跟进的评论和用户整理成列表，并写清楚判断原因。
6. 对明显无关、争吵型、玩梗型或风险高的评论做跳过记录。

## 判断规则

- 优先保留能体现真实需求、问题、求助、比较意向的评论。
- 优先保留有继续对话空间的用户，而不是只会短暂互动的用户。
- 如果评论内容和产品没有直接关系，应跳过，不要强行贴。
- 如果评论区氛围明显敏感、对立或容易引发反感，应降低介入优先级。

## 允许使用的指令

- `product.get`
- `video.get`
- `comment.list`
- `comment.scan`

## 禁止事项

- 不要在这个能力包里直接发送评论或私信。
- 不要把所有提到关键词的评论都当作有意向。
- 不要只记录用户，不记录判断依据。

## 完成标志

- 已经形成一份可交给下游使用的线索列表。
- 每条线索至少有对应的评论依据或判断理由。
- 已经说明哪些评论值得继续、哪些评论明确跳过。

## 受阻处理

- 如果视频本身相关性不足，记录原因并交回主 Agent，建议更换视频。
- 如果评论区可用线索太少，记录扫描范围与结果，再请求补充视频来源。
- 如果接口异常或风控明显，停止继续深扫并交回主 Agent。
