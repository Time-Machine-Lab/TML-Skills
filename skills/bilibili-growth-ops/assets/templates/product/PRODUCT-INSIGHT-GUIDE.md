# 产品信息提炼指引

## 作用

这份指引用来指导 Agent 对产品资料做“推广导向”的信息挖掘与提炼。

它不是固定结果模板，而是一套可持续迭代的方法说明。后续如果团队对“推广点、引流点、种子根因”的理解升级，优先更新这份指引，而不是去写死产品工作稿。

## 什么时候使用

- 新产品刚创建、资料刚导入之后
- 产品补充了新的说明、案例、素材之后
- 在创建策略（`Strategy`）和任务（`Task`）之前
- 发现当前推广判断不稳定，需要重新抽取产品核心时

## 输入范围

至少结合以下信息一起判断：

- `PRODUCT.md`
- `materials/` 下的文字资料、图片说明、案例补充
- 用户额外提供的自然语言说明

如果资料不足，要明确写出“目前缺什么”，不要硬补。

## 提炼目标

目标不是写一份产品介绍，而是沉淀出一份“后续推广可复用的判断底稿”。

这份底稿至少要帮助后续 Agent 回答清楚：

- 这个产品到底是什么
- 谁最可能对它感兴趣
- 他们为什么会在 B 站搜索、讨论或留言
- 哪些点适合公开评论承接
- 哪些点更适合继续追问或升级私信
- 哪些点一旦讲错，就容易失真、引反感或触发风控

## 核心提炼维度

以下维度是当前推荐重点，但不是死板清单。你可以根据产品类型增删、合并或重排：

### 1. 产品核心定义

- 一句话说清楚产品是什么
- 它最直接提供的结果是什么
- 它和常见替代方案最本质的差异是什么

### 2. 目标用户与意向层级

- 第一目标用户是谁
- 第二目标用户是谁
- 不同用户会如何描述自己的问题
- 哪些信号说明对方只是围观，哪些信号说明已经有较高意向

### 3. 用户问题场景

- 用户通常在什么情境下会开始关注这类产品
- 用户此时最常见的问题、卡点、顾虑分别是什么
- 用户会在 B 站用什么表达方式提这些问题

### 4. 推广点

- 哪些价值点适合公开讲
- 哪些差异点最容易被用户快速理解
- 哪些点适合在视频评论区自然承接，而不会像硬广

### 5. 引流点

- 哪些内容最容易让用户继续追问
- 哪些比较点、痛点或场景最容易引出互动
- 哪些时机适合从公开互动升级到更深一层的交流

### 6. 种子根因

这是当前最重要的一层，不要只写表面卖点。

重点追问：

- 用户为什么会开始搜索这类问题
- 用户为什么会停留、收藏、评论、追问
- 用户当前旧方案为什么不够用
- 用户迟迟不行动的真正阻力是什么
- 用户被什么样的表达触发后，才愿意继续往下走

### 7. 语言与信号

- 用户会高频使用哪些词、问题句、比较句
- 哪些词适合搜索视频
- 哪些话术更适合评论承接
- 哪些表达一看就会太营销、太假或太重

### 8. 信任证据与风险边界

- 哪些事实、案例、结果最能增强信任
- 哪些承诺不能乱说
- 哪些说法容易引发反感、争议或风控
- 当前资料里还缺少哪些关键支撑

## 产出要求

- 最终写入产品目录下的 `PRODUCT-INSIGHT.md`
- 这是一份工作稿，不要求一次成型
- 可以自由组织结构，但必须保证后续 Agent 容易复用
- 优先保留“可执行判断”，少写空泛形容词
- 如果某条判断来自推断而不是直接事实，要显式标注

## 建议写法

可以参考这种表达习惯，但不要求完全照抄：

- 先写“产品核心判断”
- 再写“目标用户 / 问题场景 / 推广点 / 引流点”
- 再写“种子根因 / 高意向信号 / 风险边界”
- 最后补“还缺什么信息”

## 不要这样做

- 不要把它写成官网介绍
- 不要只堆功能点，不追问用户为什么会行动
- 不要只写卖点，不写风险边界
- 不要为了整齐而硬套固定栏目
- 不要在资料不足时假装已经挖透
