# 任务书：运行空间与初始化底座

## 1. 任务综述

### 1.1 任务定义

建立二期统一的 `runtime-root`、初始化流程和运行目录约定。

### 1.2 核心目标

这个任务主要解决三件事：

- 系统跑在哪里
- 内容放在哪里
- 怎么初始化和检查环境

### 1.3 当前状态

- 已完成
  - `runtime-root` 的总体结构已经在设计文档中定案
  - 产品、策略、能力包、任务目录的主结构已经明确
- 未开始
  - 新系统的初始化底座还没有正式建立
  - 新系统的运行目录和巡检入口还没有落地
- 参考边界
  - 旧版运行目录、旧版状态文件和旧版工作链路不作为二期底座继续沿用

### 1.4 重要性与紧急程度

- 优先级：`P0`
- 说明：这是新系统真正落地的工程底座

### 2.2 这份任务主要产出什么

这份任务主要要把新的运行空间约定清楚：

- `db/` 放结构化事实
- `resources/` 放产品、策略、能力包和任务文件
- `exports/` 放阶段性导出结果
- 初始化时自动建立必要目录和基础文件入口

二期的运行空间不是旧版目录的再整理，而是新的工作空间。

### 2.3 本任务范围

包括：

- `runtime-root` 目录结构
- 初始化入口
- 基础目录创建规则
- 运行态健康检查和缺失检查
- 产品、策略、能力包、任务的落位规则

不包括：

- 数据库表细节
- 具体 B 站接口能力
- 具体策略内容
- 主副 `Agent` 调度实现

## 3. 注意事项

### 3.1 已知风险

- 如果运行空间不先统一，后面每一层都会各自落文件，系统会再次变乱
- 如果继续兼容旧目录，二期很容易被旧结构拖回去

### 3.2 强约束

- 二期运行空间从零建立，不继承旧版运行目录
- 详细任务内容继续走文件，不进数据库
- 所有运行态数据都在 `runtime-root` 下统一管理

### 3.3 耦合点

- 任务 3 的 `SQLite` 文件路径依赖这里
- 任务 5 的产品、策略、能力包、任务模板依赖这里
- 任务 6 的任务执行和工作记录依赖这里

### 3.4 避坑

- 不要把目录拆得过细，影响人和 `Agent` 的阅读效率
- 不要把数据库和任务文件混在同一层
- 不要把运行态文件继续写进 skill 仓库本体

## 4. 验收标准与交付物

- [ ] 二期运行态不再依赖旧版目录结构
- [ ] 后续任务可以直接基于这套工作空间继续建设
