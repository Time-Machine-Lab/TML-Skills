# bilibili-api-skill 二期主 Skill 设计

## 定位

主 Skill 就是整个 skill 的总入口和项目地图。

它是：

- agent 安装 skill 后第一眼看到的入口
- 用户了解整个 skill 的基础入口
- 整个项目的导航页和路由页

主 Skill 的核心职责只有一个：

- 把用户和 agent 导到正确的下一层文档和入口

## 主 Skill 要做什么

主 Skill 需要清楚说明 5 件事：

1. 这个 skill 是做什么的
2. 整个 skill 分成哪几层
3. 用户第一次应该怎么开始
4. agent 第一次应该怎么开始
5. 当前目标不同，应该跳去哪里

## 主 Skill 里应该放什么

我建议主 Skill 只放下面这些内容：

### 1. Skill 简介

用最短的话说明：

- 这个 skill 是做什么的
- 它解决什么问题
- 它大概怎么工作

### 2. 分层地图

把当前结构讲清楚。

建议至少说明这些层：

- `Command`
- `Capability`
- `Strategy`
- `TaskSpec`
- `Domain`
- `OperationRecord`
- `runtime-root`

### 3. 使用入口

分别说明：

- 用户怎么开始
- agent 怎么开始

### 4. 路由规则

说明不同场景下应该先去哪里。

比如：

- 未初始化
- 需要创建产品
- 需要选择策略
- 需要创建任务
- 需要执行任务
- 需要查历史记录

### 5. 阅读顺序

给出一条默认阅读顺序，让人和 agent 都不会迷路。

## 主 Skill 不应该放什么

主 Skill 不应该变成大杂烩。

所以这里不放：

- 完整指令手册
- 完整策略细节
- 完整任务内容
- 过多运行细节

这些内容都应该放到对应的下层文档里。

## 主 Skill 和其他文档的关系

主 Skill 负责导航，其他文档负责展开。

关系可以直接理解成：

```text
主 Skill
  -> 运行环境文档
  -> 领域实体文档
  -> 指令包文档
  -> 编排对象文档
  -> 具体策略
  -> 具体任务
```

## 结构

主 Skill 结构收成这 6 段：

1. 这个 skill 是做什么的
2. 分层地图
3. 用户怎么开始
4. agent 怎么开始
5. 不同目标怎么路由
6. 推荐阅读顺序

## 定案

- 主 Skill 是整个项目的总入口和项目地图
- 它的核心职责是导航，不是展开所有细节
- 它需要帮助用户和 agent 快速找到正确入口
- 它只负责说明结构、入口、路由和阅读顺序
