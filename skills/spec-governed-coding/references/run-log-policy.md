# Run Log Policy

Run logs are task-level reflection artifacts, not continuous bookkeeping.

## Write a run log when

- a task is being used to validate the workflow
- the task involved meaningful routing or delegation choices
- a task produced surprising waste, rework, or quality issues
- the team wants evidence before changing the governance protocol

## Skip a run log when

- the task was tiny
- routing was trivial
- no meaningful governance decision was made

## What a run log should capture

- what stayed with the master
- what was delegated
- where waste appeared
- whether the workflow felt worth reusing

It does not need exact token accounting.
